# v0.3.5 计划: 计费周期 + 到期倒计时 + 流量重置

## 核心变更

- **删除 `reset_day` 概念**（前端 localStorage、后端都没有）
- **新增 `expires_at` + `billing_period`**：agent 配置上报，server 持久化
- **倒计时 + 流量清零完全同步**：同一个日期承担续费提醒 + 流量周期双重角色
- **卡片 sub 行显示纯数字倒计时**：如 `23`，hover 显示 `到期: 2026-08-15`

## 计算逻辑

```
expires_at = "2026-07-15", billing_period = "1m"

date       show  notes
June 29    16    距离 July 15
July 14    1     明天到期
July 15    31    跳到下个周期 Aug 15, 流量归零
Aug 14     1
Aug 15     31    跳到 Sep 15, 流量归零
```

`calcNextReset()`: 从 anchor 开始按 period 向后跳到第一个 > now 的日期，`days = ceil(差)`

### 9. `scripts/install-agent.sh`
- TCPing 问答之后、生成 agent.yaml 之前插入:

```bash
echo
echo "VPS billing setup (for dashboard countdown and traffic reset):"
echo "  1) Monthly (1m)"
echo "  2) Quarterly (3m)"
echo "  3) Semi-annual (6m)"
echo "  4) Annual (12m)"
echo "  0) Skip"
read -rp "Select billing period [0]: " PERIOD_CHOICE < /dev/tty

EXPIRES_AT=""
BILLING_PERIOD=""
case "$PERIOD_CHOICE" in
  1) BILLING_PERIOD="1m";  DEFAULT_EXPIRY=$(date -d "+1 month" +%Y-%m-%d) ;;
  2) BILLING_PERIOD="3m";  DEFAULT_EXPIRY=$(date -d "+3 months" +%Y-%m-%d) ;;
  3) BILLING_PERIOD="6m";  DEFAULT_EXPIRY=$(date -d "+6 months" +%Y-%m-%d) ;;
  4) BILLING_PERIOD="12m"; DEFAULT_EXPIRY=$(date -d "+1 year" +%Y-%m-%d) ;;
  *) BILLING_PERIOD="" ;;
esac

if [ -n "$BILLING_PERIOD" ]; then
    read -rp "Next renewal date [${DEFAULT_EXPIRY}]: " EXPIRES_AT < /dev/tty
    EXPIRES_AT="${EXPIRES_AT:-$DEFAULT_EXPIRY}"
fi
```

- agent.yaml 生成模板: 在 `insecure: false` 后加 `billing_period: "${BILLING_PERIOD}"` 和 `expires_at: "${EXPIRES_AT}"`
